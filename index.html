HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The html tag represents the start of the html document
  • The order of headers are importan, always starts with h1
  • Best practice is to use semantic elements when possible

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • There are 3 ways to use CSS: inline CSS, internal CSS style sheet, or external CSS style sheet
  • The * selector is a wild card symbol, which will apply the CSS rules to all elements on the page
  • When applying CSS rules from an external CSS style sheet, the style sheet needs to be linked to the HTML file by using the link tag in the head element

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switched to it
  • git pull origin main: pulls from the remote main branch and commits updates to your current working branch
  • git add -A: to stage all changed files
  • It is good practice to commit often, keeping each task separate

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • Control flow can be interupted with the use of conditional statement and loops
  • JavaScript filed should be linked to HTML files right beforoe the closing body tag
  • Variable and function are case sensitive and should be unique
